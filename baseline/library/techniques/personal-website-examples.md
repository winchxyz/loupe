# Personal website archetypes — applied build decisions

Core idea: a personal site's structure follows its ONE goal (hire me / show work / inform guests / drive a signup). Pick the archetype that serves that goal first, THEN style it. Don't pour every idea into one page — the goal dictates which sections exist and which visual register to use.

## Reusable patterns (each = a structure + when to reach for it)

1. **Minimalist resume / CV.** Lead with name + title + key achievements in a strong visual hierarchy; carry the page on generous white space and a modular section stack (skills, projects, history) rather than chrome. Use when the goal is "be hired / be read fast" and the audience scans. Choose a high-readability text font; let typography, not decoration, do the work.

2. **Work-speaks portfolio (clean grid).** A restrained grid of high-resolution work images + concise project blurbs; add short "process / behind-the-scenes" notes to deepen credibility. Use when the work itself is the argument (designers, devs, makers). Keep the frame neutral so the images carry contrast; verify it holds across breakpoints.

3. **Immersive visual portfolio (full-bleed imagery).** Full-screen images, minimal text, dead-simple nav — visuals do the talking. Use ONLY for highly aesthetic, visual-first fields (model, photographer, art-director) where strong imagery exists. Keep descriptions tiny and in a plain professional font; keep contact one tap away. Avoid when the work is non-visual — emptiness reads as thin, not premium.

4. **Editorial / visual-essay (magazine layout).** Full-width layouts + curated visuals + long-form text for essays, case studies, thought leadership. Use for writers and content creators who sell through prose. Establish ONE visual theme (color + type) up front and hold it; add social-share affordances to extend reach.

5. **Accessible community / event site.** High-contrast text, simple navigation, flexible content blocks (schedule, speaker/performer bios, galleries), and explicit CTAs. Use for events, clubs, org pages where reach and inclusion matter more than flourish. Accessibility is the design here, not an afterthought — bake in contrast and device-testing. Add practical conversion aids (QR check-in, RSVP).

6. **Occasion / informational site (e.g. wedding).** Mood-led palette + organic elements wrapped around hard logistics (RSVP, travel, accommodations, gifts). Use when the job is "look lovely AND answer practical questions." Balance atmosphere against utility — never let the dreamy styling bury the details people actually came for.

7. **Rule-breaking signature style (retro / industrial / etc.).** A distinctive register — geometric shapes, bold palette, characterful display type — to be memorable. Use when standing out IS the goal and the owner has a clear personal style. Break enough rules to be memorable while keeping the layout functional; distinctiveness must not cost legibility or navigation.

## Cross-cutting rules
- **Match register to field.** Corporate/tech → structured + restrained with a personality bump in the bio; creative/visual → imagery-forward. Never default every personal site to the same minimalist template.
- **Decide content by goal, not by habit.** Candidate sections: biography, work samples, writing, contact, resume/CV, testimonials. Include only what serves this site's goal; promoting an event ≠ showing a writing portfolio.
- **Balance text with visuals via negative space** to steer the eye; authenticity (real colors/fonts/photos that fit the person) beats generic polish.
- **One clear CTA per goal** — make "reach me / RSVP / hire me" obvious and easy to find.

## Build checklist
- [ ] Named the site's ONE primary goal before choosing a layout?
- [ ] Picked the archetype that serves that goal (resume / grid / immersive / editorial / community / occasion / signature)?
- [ ] Included only goal-relevant sections (bio, work, writing, contact, resume, testimonials) — nothing padded in?
- [ ] One consistent visual theme (palette + type) established and held across the page?
- [ ] Layout verified across devices, with contrast/accessibility checked, not assumed?
- [ ] Exactly one obvious CTA, with contact info easy to find?
- [ ] Visual register matches the owner's field and reads as authentically theirs?
