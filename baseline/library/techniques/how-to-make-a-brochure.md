# Brochure / folded-print layout — applied rules

Core idea: a brochure is a layout whose STRUCTURE is the fold. Information flows panel-by-panel in a fixed reading order, each panel does one job, and key messages must never straddle a crease. The discipline (fold-first planning, one-message-per-panel, obvious CTA, print-safe export) transfers directly to any sectioned/scroll-as-fold web layout.

## Decisions before you design
1. Fix audience + purpose FIRST — informational / promotional / educational. This decision drives tone, panel count, and content hierarchy; do not start the visual layout until it's named.
2. Choose the fold STRUCTURE before content, because the fold dictates reading order and panel count. Pick deliberately:
   - **Bi-fold** (4 panels: front cover / back cover / 2 inside) — use for image-led or minimal-text pieces: invitations, menus, simple programs. Cheapest, simplest.
   - **Tri-fold** (6 panels, C-fold) — use as the default workhorse: sequential/step-by-step content, price lists, small product showcases, anything mailer-sized. Right panel folds in first, then left over it to become the cover.
   - **Gatefold** (two small outer panels open like doors onto one large center panel) — use for a surprise reveal / high-impact hero visual: product launches, special promos. Reserve the big center panel for the single strongest image.

## Layout rules (each = a build decision)
3. One message per panel — map content to panels before styling; position key messages where they won't get lost in a fold/crease. (Web parallel: don't let a headline straddle a section break or sticky-pin boundary.)
4. Lead with strong, concise copy — less is more. Cut the intro paragraph to one or two short blocks (see template 12: one intro + two short paragraphs carried the whole piece).
5. Break dense text up with headings, bullets, and lists; never ship a wall of body copy on a panel.
6. Build a clear visual hierarchy that guides the eye through the panels in fold order (zig-zag and sequential flows both work — template 8 uses an explicit zig-zag).
7. Use negative space deliberately — give every element room to breathe; cramming kills the print feel.
8. Lock ONE consistent color palette + brand styling across every panel and both sides. Recurring shapes/colors (logo colors echoed as large shapes, a wave motif repeated front and interior) create cohesion — reuse a motif, don't invent per panel.
9. Convert complex data to charts/graphs (donut charts, stats callouts) instead of paragraphs — keeps panels clean and scannable.
10. Make the CTA unmissable and literally actionable: "visit URL", "call number", "sign up", plus a QR code near contact details. Put contact info on more than one panel (e.g. an inside page AND the back) so it's findable whenever the reader is ready.
11. Prioritize legibility/accessibility — easy-to-read fonts, inclusive choices; never trade readability for a flourish.

## Type + palette patterns worth reusing
12. Match the type system to the brand energy:
    - Clean/corporate trust → a single humanist sans (Poppins, template 8).
    - Sleek/futuristic/tech → geometric display (Orbitron, template 11) paired with hexagon/grid motifs.
    - Edgy/youthful/event ('80s + Y2K neon) → a loud multi-font mix (Phosphate, Quantico, Press Start 2P, Plus Jakarta Sans, etc.) to manufacture energy AND establish hierarchy. Reserve this only when the brand is genuinely playful — a multi-font stack is a deliberate effect, not a default.
13. Cheerful-modern palette that reads well in print: cream + teal + orange (template 3). Pair circular photos/shapes with the logo for a friendly, cohesive feel.

## Print/export safety (porting to "ship the artifact" discipline)
14. Design both sides in ONE file; align both sides to the real fold geometry so nothing is cut off or out of order.
15. Set the canvas to a real brochure size: default 8.5 x 11 in (classic tri-fold); go 8.5 x 14 or 11 x 17 when content needs more room. Keep trim + fold lines visible while designing.
16. Before export, verify color mode, bleed, margins, and format; export print as a high-res PDF with crop marks and bleeds. Always proofread + run a test print to check alignment, folds, and image quality before committing.

## Build checklist (tick while applying)
- [ ] Audience + purpose named, and the fold type chosen to match BEFORE styling?
- [ ] Content mapped one-message-per-panel, with no key element crossing a fold/section break?
- [ ] One palette + one recurring motif carried across every panel and both sides?
- [ ] Dense info turned into bullets/charts, with generous negative space?
- [ ] CTA obvious + actionable, contact info + QR present on at least two panels?
- [ ] Type system matched to brand energy (one trusty sans vs. a deliberate multi-font mix)?
- [ ] Canvas at a real size with bleed/margins/color-mode verified, and a test render proofed end-to-end?

> Ties to our experience: rule 3 (key content must not straddle a fold) is the print twin of our sticky-pin responsive sections — a headline that crosses a pin boundary reads broken the same way. Rule 16 (verify the real export, proof a test render, don't trust the on-screen frame) is exactly our PAIN.md verification-coverage theme: trusting a settled frame / a clipped screenshot instead of checking the actual shipped output.
