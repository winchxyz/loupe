# Call-to-action patterns — applied rules

Core idea: a CTA is a designed prompt for an immediate response, not just a styled button. Pick the underlying PATTERN that fits the page's job, then make it legible with contrast, copy, and placement. Below are reusable patterns (the mechanism, not the brand names) — each with a "when to use."

## Reusable CTA patterns

1. **Multi-placement journey (top / mid / oversized bottom)** — Repeat the same primary CTA at the top, middle, and an extra-large version at the page end so the user can convert at any scroll depth. *When:* long scroll/landing pages where the decision builds over the page.
2. **Progressive CTA specificity** — Open broad ("Get started"), then narrow as the user learns more ("Get started with X"). The copy tracks rising intent. *When:* multi-section product pages where the visitor's goal sharpens as they scroll.
3. **Offer-led / urgency CTA** — Lead with the concrete offer in the button or lead-in ("Get Offer", a countdown, a seasonal deal) to tap savings + scarcity. *When:* promos, limited deals — but keep urgency honest so it doesn't read as forced.
4. **Free-trial / no-risk CTA** — State the free trial, free plan, or "no credit card required" right at the CTA to remove the last barrier. *When:* SaaS sign-ups and any paid product with a trial.
5. **Sticky / persistent CTA** — A button that follows the scroll (header bar or a corner) stays available without interrupting discovery. *When:* key single actions like Sign Up / Start Watching where you want constant access, not nagging.
6. **Action-specific copy** — Replace generic verbs with the exact action: "Shop new", "Shop best-sellers", "Book a demo", "Customize Now", "View Docs". Specificity removes guesswork. *When:* almost always; default to specific over "Shop"/"Submit"/"Click here".
7. **Choice pair (side-by-side intents)** — Two CTAs serving different intents (e.g. primary "Sign up free" + secondary "Discover the possibilities" / direct purchase). Give the primary clear visual weight; the secondary is lower-emphasis. *When:* audiences split between "buy now" and "still exploring".
8. **Context-backed CTA (answers "Why click?")** — Surround the button with lead-in copy stating the benefit, plus trust signals: ratings, "no credit card required", social proof. *When:* higher-commitment asks (demos, paid sign-ups) where doubt blocks the click.
9. **Well-timed pop-up / slide-in** — Fire at a natural engagement point, give an obvious X to dismiss, and earn the interruption (a countdown or prize like "Enter to win" raises motivation). *When:* newsletter capture and contests — never on immediate page load if it harms UX.
10. **Form-capture CTA** — The button opens a short form that captures qualifying details (company type, etc.) so the response can be personalized. *When:* B2B / tailored onboarding where segmentation pays off.
11. **Soft "Read more" / "Learn more"** — A low-clutter indirect CTA that moves the user one step deeper. Let surrounding white space carry the emphasis. *When:* editorial/content pages, or as the secondary next to a hard primary.
12. **On-brand CTA** — Match copy to brand voice and even use the brand/product name ("Get your headspace", branded sign-in). Prioritize clarity over cleverness; a clear plain CTA beats a clever opaque one. *When:* the brand voice is a real asset — but never at the cost of the user understanding what happens on click.

## Make any CTA legible (the always-on rules)
- **Contrast + white space:** give the CTA bold color/typography against generous surrounding white space so the eye lands on it; size it proportional to the screen — big enough to spot, not so big it feels unnatural.
- **Above the fold + hierarchy:** put the most important CTA above the fold and in high-traffic zones; one clear primary per view, others demoted.
- **Actionable verbs:** strong, concise verb phrases so the user knows exactly what happens on click.
- **Bold / all-caps sparingly:** an all-caps word like FREE can draw the eye and signal urgency — use as an accent, not for the whole button.
- **Hover/interactive states:** add hover feedback for function and delight (especially on a primary button).
- **Test and iterate:** A/B test color, placement, and copy; keep the variant real user behavior prefers.

## Build checklist
- [ ] One clear PRIMARY CTA per view, above the fold, with a demoted secondary if intents differ?
- [ ] Copy is action-specific (not "Shop"/"Submit"/"Click here") and matches the user's current intent/scroll stage?
- [ ] Strong contrast + white space + correct size make it the obvious focal point, with a hover state?
- [ ] Friction removed at the button (free / no-card / rating / social proof) and any urgency is honest?
- [ ] Sticky or repeated placement gives multiple convert points on long pages?
- [ ] Pop-ups fire at a real engagement moment with an obvious dismiss?
- [ ] Verified the rendered button (contrast ratio, hover, sticky behavior) in the real preview, not just the source?
