# Futuristic fonts — applied build decisions

Core idea: "futuristic" type is almost always a DISPLAY job — sharp/geometric/pixel/segmented letterforms that carry tone at large sizes and fall apart at small ones. Choose by the SUB-FLAVOR you want, set it big, keep it short, and pair it with a quiet workhorse for body. Don't decorate with two of these at once.

## Pick by flavor (each = a reach-for-this decision)

1. **Clean geometric sci-fi (space-age, HUD, "innovation" brand).** Default reach: **Orbitron** (rounded corners + angular, classic sci-fi/HUD, multi-weight, stays legible) or **Tomorrow** (minimalist sci-fi glyphs). Also: **Michroma** (technical-drawing feel; uniquely holds up small enough for app UI), **Zen Dots** (legible at both large AND small — safest of the group), **Krona One** (wide, assertive, impact without losing legibility). Use for headlines, hero titles, logos, packaging.

2. **Industrial / blocky / squared-off.** Reach: **Tektur** or **Nova Square** (sharp angular, monolinear, industrial), **Electrolize** (squared modular, reads clean on screen), **Russo One** (bold rounded-but-robust, billboards). Use when the brand wants "engineered/machined," not "spacey."

3. **Variable font that DOES something.** Reach: **Unbounded** (geometric + humanist curves, broad language support → good when you need warmth + global scripts), **Workbench** / **Sixtyfour** (variable weight/width PLUS animatable scanlines & pixel-bleed → CRT/retro-computer motion), **Climate Crisis** (weight axis = a data narrative; pick for time-stamped/urgency visuals only). Reach here when you want one face to flex or animate across an axis.

4. **Pixel / bitmap / 8–16-bit nostalgia.** Reach: **Pixelify Sans**, **DotGothic16** (16x16 bitmap, highest readability of the pixel set), **Workbench** (Amiga), **Sixtyfour** (C64). Short copy only — retro/gaming/pixel-art energy. Never body text.

5. **Mono / terminal / "code & data" tech feel.** Reach: **Space Mono** (1960s space + early-computer, the versatile pick — handles short body too), **Syne Mono** (typewriter/terminal). Use for technical docs, data displays, code-themed sections, and as a tasteful body/label pairing for the louder display faces above.

6. **Segmented / digital-display (LED, calculator, clock).** Reach: **Audiowide** (LED-style, rounded blocky), **Digital Numbers** (calculator/clock segments). Great for counters, stats, HUD numerics — display size only.

7. **Distressed / glitch / cyberpunk edge.** Reach: **Rubik Glitch** (distortion, cyberpunk), **Rubik Broken Fax** (crystal/chip disruption), **Rubik 80s Fade** (synthwave halftone), **Wallpoet** (stencil/rebellious). Large sizes, posters/album covers/digital art; legibility intentionally compromised, so headlines/logos only.

8. **Retro Deco / stylized vintage-future.** Reach: **Monoton** (all-caps Art Deco, uniform stroke — sparingly, big), **Geo** (1920s Bauhaus-experimental geometric), **Train One** (TRON-poster outlined letterforms with depth). Use for branding moments with a period-future twist.

9. **Friendly / approachable "soft future."** Reach: **Smooch Sans** (tall, rounded, welcoming — younger/kids audiences), **Gugi** (Hangul-derived organic flow), **Turret Road** (low contrast, high x-height — rare futuristic face actually OK for body text and mobile UI), **Doto** (large x-height, works in body). Reach here when "futuristic" must not read cold/aggressive.

## Pairing & usage rules (apply every time)

- **One decorative face per design.** Let the display font do the heavy lifting; pair it with a plain sans or a mono (Space Mono / Syne Mono) for body and labels. Never stack two glitch/pixel/deco styles.
- **Cap the size floor.** Reserve flavors 4/6/7/8 for headlines, hero, logos, numerics — switch to a clean body font (or flavor 1/9's legible members: Zen Dots, Turret Road, Doto, Michroma) for anything paragraph-length.
- **Match font to vibe, not to trend.** Sleek-minimal vs edgy-experimental is a real fork — pick the sub-flavor that matches theme/tone, don't default to "Orbitron because sci-fi."
- **Tune kerning + leading deliberately.** Tighten for sleek/precise; loosen for open/digital — but never past readability.
- **Verify across media & sizes.** Many of these lose clarity in print or at small sizes; render the actual headline at the actual size and check — don't trust the specimen.

## Build checklist (use when choosing futuristic type)
- [ ] Picked the sub-flavor (1–9) that matches the project's tone, not just "futuristic"?
- [ ] Only ONE decorative display face; body set in a quiet sans/mono?
- [ ] Display face used at headline size only — body in something legible?
- [ ] Kerning/leading tuned to the desired feel without breaking readability?
- [ ] Rendered the real headline at the real size and (if printed) test-printed?
- [ ] If using a variable/animated face (Workbench, Sixtyfour, Climate Crisis), is the axis actually carrying meaning, not just novelty?
