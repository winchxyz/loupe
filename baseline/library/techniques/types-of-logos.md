# Logo types — pick the form, then build it

Core idea: a logo is one of 12 structural forms, not a freestyle drawing. Pick the form from the brand's personality + where the mark has to live (favicon, nav, hero, loader), then design within that form. Each pattern below = a build decision with a "reach for it when".

## The patterns (each = a form + when to reach for it)

1. **Abstract mark** — geometric shapes/lines, no literal object; lean on Gestalt (similarity, proximity, continuity) so the eye completes the form. Reach for it when: tech / consulting / fintech / wellness that wants "modern & conceptual" and has no obvious object to depict. Pair with a wordmark for context.
2. **Line-drawing mark** — one continuous unbroken stroke, or a few clean separate strokes. Reach for it when: you need a mark that holds up tiny and scales from favicon to billboard; fashion, minimalist tech, creative agencies. Choose stroke-based SVG so weight stays crisp at any size.
3. **Monogram** — brand initials as the mark. Reach for it when: the full name is long but initials are distinctive; luxury, heritage, family business, boutique hospitality. Our TESQ "TES + square Q" lockup is exactly this pattern.
4. **Letterform** — a single stylized letter (vs. initials). Reach for it when: one letter can carry the brand alone (startups, media, F&B) and you want a symbol + text hybrid in one glyph.
5. **Wordmark / logotype** — the full name as custom-set type, no separate icon. Reach for it when: the name itself is short and worth reinforcing; SaaS, professional services, retail. Differentiate via custom letterforms (Netflix arch) or a slant for motion (Visa) — not a stock font.
6. **Pictorial mark** — one recognizable symbol, text-free (Apple, Target). Reach for it when: the brand is established enough that the symbol alone reads, and you need a stand-alone icon for app tiles. Must survive at any scale.
7. **Combination mark** — symbol + wordmark as a flexible lockup. Reach for it when: you're a newer brand needing both recognition AND legibility; this is the safe default because it degrades to either piece. Build it as a system: horizontal lockup, stacked lockup, icon-only.
8. **Emblem / badge** — identity sealed inside a frame/border (crest, shield). Reach for it when: tradition / heritage / institutions / clubs. Avoid for tiny UI — interior detail dies at favicon size.
9. **Mascot** — a character/creature with personality. Reach for it when: F&B, kids, sports, gaming — you want emotional warmth and merch-ability, especially for younger audiences.
10. **Animated logo** — a still mark plus motion (color flow, build-on). Reach for it when: digital-first brands wanting a memorable loader / intro; media, gaming, startups. Design the static state first, then layer motion — never depend on motion to be legible.
11. **3D logo** — depth/render giving realism or futurism. Reach for it when: gaming, automotive, architecture, luxury wanting a "striking, premium" feel. Keep a flat 2D fallback for small sizes and print.
12. **Dynamic logo** — adapts across context/platform/season/interaction (Spotify waves, Meta variants). Reach for it when: a brand system, not a single fixed mark — define the constant core + the variable elements explicitly so it stays recognizable while it shifts.

## Universal build rules (from the article's own design section)
- **Scale-test at favicon size first.** If detail dies at app-icon scale, simplify — don't ship a mark that only works in the hero.
- **Design the monochrome version.** Always test in black-and-white; a logo that needs color to read is broken.
- **Demand strong contrast.** Avoid low-contrast pairings (e.g. light-blue on white) that go flat; keep mark and background clearly distinguishable.
- **Readability over ornament.** Ornate letterforms hurt legibility small — pick clean, legible type that still matches personality.
- **Build it as a system, not one PNG.** Multiple formats (print + digital), multiple sizes, color variations — bake versatility/scalability in from the start.

## Build checklist
- [ ] Picked ONE of the 12 forms on purpose (brand personality + where the mark lives), not a freestyle drawing?
- [ ] Renders clean at favicon / app-icon size (scale test done)?
- [ ] Monochrome version exists and still reads?
- [ ] Mark vs. background contrast is strong, not flat?
- [ ] If animated/3D/dynamic: a static, legible fallback ships for small sizes?
- [ ] Delivered as a lockup system (horizontal / stacked / icon-only), not a single fixed file?

> Ties to our experience: TESQ is a monogram (Type 3) — "TES + a square filled Q" — and the favicon/loader scale test is exactly the discipline our matte-glass loader work needs; the article's "test in monochrome / at icon size" maps directly to our verification-coverage gaps (checking the wrong state, trusting one settled frame).
