# Presentation & narrative design — reusable patterns

The source is a 60-item roundup. The point is NOT the 60 tips — it's the small set of underlying patterns that survive when you strip the brand names. Each pattern below = a build decision plus a "when to use." Most transfer directly to a scroll-told site (our 02b journey is exactly this: a sequence of screens, one idea each).

## Reusable patterns

1. **One idea per surface.** Cap each slide/section to a single message; highlight one critical element. When to use: any time a screen starts carrying two competing points — split it. (Tips 43, 41.) Corollary 5/5/5 rule from the FAQ: ≤5 bullets, ≤5 words each, ~5 min of attention — a hard density ceiling, not a suggestion.
2. **Strong-open hook.** Lead the sequence with ONE of: a provoking question, a powerful quote, or a concrete statistic/fact — never a title card alone. When to use: the first screen, where you either earn attention or lose it. (Tips 19, 49, 55.) For a quote, verify original context before shipping it.
3. **Bookend the arc.** Open with a hook, close with a concise summary + an explicit single call-to-action. When to use: the last screen — state the one next step (visit / schedule / contact), don't trail off. (Tips 18, 28.)
4. **Narrative spine.** Order screens as a five-part arc: intro → rising action → climax → return → conclusion. When to use: whenever the sequence is more than a flat list — let structure, not decoration, carry the viewer. (Tips 30, 40.) This is literally our 02b water→space→sun journey.
5. **Wayfinding for long sequences.** Add a contents/overview screen up front and let structure stay legible after the fact (shareable). When to use: decks/sites past ~7 sections, or anything the viewer revisits unguided. (Tip 27.)
6. **Constrained palette, accent for signal.** Default to a two-tone or neutral base, then reserve ONE bright accent exclusively to mark the thing that matters (a stat, a CTA). When to use: every layout — make color earn its place instead of decorating. (Tips 10, 11, 2.) Accent is a pointer, not a paint job.
7. **Color carries meaning, accessibly.** Pick hues by intent — blue = trust/reliability, yellow = creativity/positivity — and check chart/UI color pairs against color blindness. When to use: data viz and any emotion-led screen; never rely on hue alone to encode a distinction. (Tips 15, 4.)
8. **Type sets tone, consistently.** Hold one type system across the whole sequence; break it only to emphasize. Match the face to the message: serif = modern/refined, handwritten = personal. When to use: from the first screen — wandering fonts read as broken, not lively. (Tips 12, 5, 1.)
9. **White space is a signal, not waste.** Use emptiness to point the eye at the focal element. When to use: any screen that feels busy — remove before you add. (Tip 51.)
10. **Dividers to segment dense content.** On text- or data-heavy screens, use lines/rules to draw boundaries around ideas. When to use: complex graphs or multi-point screens that read as a wall. (Tip 7.)
11. **Visual hierarchy by deliberate layout.** Compose so the eye lands on the most important element first; don't distribute weight evenly. When to use: every screen — hierarchy is the layout's job, not the reader's. (Tip 36.)
12. **Motion that guides, not decorates.** Use subtle animation/transitions to direct attention to the next key point. When to use: emphasis and handoff between ideas — reject motion that doesn't steer the eye. (Tips 3, 26.)
13. **Show, don't tell — live artifacts.** Embed a working prototype, demo, screencast, or annotated screenshot instead of describing it. When to use: any product/process claim that a real artifact proves faster than prose. (Tips 24, 35.)
14. **Right visual for the data shape.** Match form to content: timeline/Gantt for sequence-over-time, diagram/flowchart for process, map for geography, infographic for dense multi-stat stories, mind map for relationships. When to use: pick by the data's structure, not by what looks nicest. (Tips 22, 50, 48, 53, 60.)
15. **Original over stock imagery.** Use custom photography/illustration that forms its own narrative series; avoid generic stock. When to use: hero and story screens where a templated look would undercut credibility. (Tips 8, 25.)
16. **Audience-fit pass.** Tune depth, tone, and examples to the actual viewer; use relatable analogies (sports, weather) for abstract ideas. When to use: a final editing pass over copy before ship. (Tips 31, 20.)

(Interaction/delivery tips — Q&A breaks, gamification, props, body language, pacing, surprise guests, AR — are live-performance ideas, out of scope for a built site; noted but not patterned here.)

## Build checklist

- [ ] Does each screen carry exactly one idea (and obey a density ceiling like 5/5/5)?
- [ ] Does the FIRST screen hook with a question/quote/stat, not just a title?
- [ ] Does the sequence form an arc (intro → climax → conclusion) and end on one explicit CTA?
- [ ] Is the palette constrained, with the bright accent reserved ONLY for the thing that matters?
- [ ] Is the type system consistent, with deviations used only for deliberate emphasis?
- [ ] Is each data point shown in the form that fits its shape (timeline / diagram / map / infographic)?
- [ ] Is there enough white space pointing the eye at each screen's focal element?

> Ties to our experience: our 02b/TESQ build IS a five-part narrative spine (water → space → sun), so patterns 1–4 (one idea per screen, strong open, bookended arc, narrative spine) are the structure, not a metaphor. Pattern 6 (constrained palette + single signal accent) and pattern 11 (deliberate hierarchy) map to the matte-glass UI where one accent must read against a busy WebGL background — and to the PAIN.md stagger/table bugs, where uniform visual weight either lied about grouping or broke column continuity.
