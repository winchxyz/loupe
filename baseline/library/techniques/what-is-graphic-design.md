# Graphic design fundamentals — applied rules

Core idea: design is clarity first, aesthetics second. Every visual element must guide attention, carry the story, or make information easier to parse. If an element does none of those, cut it. Use these as build decisions, not vocabulary.

## The element rules (each = a build decision)
1. Color — choose a palette for the EMOTION and message, not for decoration. Pick one mood (calm, energetic, premium) and let it drive accents; don't introduce a hue that doesn't earn its place.
2. Texture — add texture/grain only to create depth or shift the feel deliberately. Avoid texture on text-bearing surfaces where it costs legibility.
3. Typography — control font, size, weight, AND arrangement together; readability comes before flourish. A bigger type-scale jump reads as hierarchy faster than a color change.
4. Line — use lines (rules, dividers, alignment edges) to define space and steer the eye. A strong shared baseline does more grouping work than boxes.
5. Shape — use geometric vs. free-flowing shapes to create contrast and rank elements; matching shapes read as a set, an odd shape reads as the focal point.
6. Space — treat negative space as an active tool: more empty area around an element promotes it. Don't fill space; spend it on the one thing that matters per view.
7. Balance — distribute visual weight on purpose. Symmetry reads as stable/formal; asymmetry (size, color, texture variation) reads as dynamic — pick one and commit per section.
8. Harmony — give all elements a shared characteristic (a type family, a corner radius, a spacing rhythm) so the page reads as one identity, not assembled parts.
9. Visual hierarchy — establish exactly ONE clear focal point per view (hero line, primary CTA) and let everything else defer to it. Two co-equal focal points = no focal point.
10. Storytelling over ornament — let color, icon, illustration, and layout carry the narrative; a single symbolic image beats a paragraph. Decoration that says nothing is cut.
11. Consistent branding — reuse the same logo, palette, type, and radius across every touchpoint; repetition is what builds the trust and recall, not novelty per screen.
12. Performance is design — compress/serve images in the right format and keep layouts responsive; a fast, mobile-correct page is part of the visual result, not separate from it.

## Build checklist (use when reviewing a composition)
- [ ] Does every element guide attention, tell the story, or clarify info — or is it just decoration to cut?
- [ ] Exactly one focal point per view (one hero / one primary CTA), everything else deferring?
- [ ] Palette tied to a single intended emotion, no orphan hues?
- [ ] Type hierarchy carried by scale/weight/arrangement, legible before pretty?
- [ ] Negative space spent on the important element, not filled edge-to-edge?
- [ ] Balance chosen deliberately (symmetric=stable / asymmetric=dynamic), consistent within the section?
- [ ] Shared characteristics (type, radius, spacing rhythm) so the page reads as one identity?
- [ ] Images compressed + responsive so the design stays fast and mobile-correct?

> Ties to our experience: the "one focal point / shared characteristics" rules map to PAIN.md — the per-cell stagger that read as crooked broke common-fate harmony, and the repos-table header misalignment broke the column grid's similarity/continuity. On 02b the glassmorphism UI works because radius, blur, and type stay shared across every pinned section (harmony), while each scroll chapter owns a single focal beat (water/space/sun).
