# Display type for thumbnails & image overlays — applied rules

Core idea: a thumbnail is a 1280x720 rectangle that has to win a click in a split second. The job of the typeface is to hold up at *any* size, survive busy/visual backgrounds, and stay on-brand. The same constraints apply to any large headline laid over imagery or motion. Pick weight and family for IMPACT + LEGIBILITY first, personality second.

## Pick by the job (named recommendations)

### High-impact / stop-the-scroll headlines (go heavy, all-caps friendly)
- **Anton** — thick blocky sans, maximum presence; default for action/high-energy (fitness, sports, gaming, motivational).
- **Bebas Neue** — tall condensed all-caps display; sleek, high-contrast, packs many letters into a tight width. Tech/gaming/tutorial.
- **Jaro** — bold variable display with built-in optical sizing, so it stays sharp scaled up or down; sports/gaming/events.
- **Holtwood One SC** — heavy vintage block letters, poster-style; outdoor/automotive/DIY/food.
- **Asset** — bold condensed *serif* display, industrial/technical edge. Headlines/logos ONLY — never body text.
- Avoid: thin or overly decorative faces here — they dissolve into the background at thumbnail scale.

### Clean modern / "professional but punchy" (versatile weighted sans)
- **Montserrat**, **Poppins** — geometric, friendly, readable at any size; safe go-to for lifestyle/travel/fashion/education/marketing.
- **Inter**, **Geist Variable**, **Albert Sans** — screen-native, minimalist, Swiss-flavored; tech/software/interfaces/editorial. Geist is variable (tune weight AND width per layout).
- **Archivo**, **Rubik** — wide weight range, punchy at bold; tech/sports/news/finance (Archivo), friendly/lifestyle (Rubik, rounded corners).
- **Rethink Sans** — DM Sans/Poppins lineage with OpenType features; corporate/editorial/UI.
- Rule: with these, use a **heavier weight** for overlay text. **Raleway** is elegant but MUST go heavy on small screens or it vanishes.

### Friendly / playful / rounded (warmth, approachability)
- **Nunito**, **Quicksand**, **Concert One** — rounded terminals, soft, on-screen optimized; food/lifestyle/wellness/kids/creative.
- **Bangers** — comic display, fun and loud; comedy/gaming/reaction.

### Serif / editorial (sophistication, "serious" or luxury)
- **Instrument Serif** — high-contrast modern serif, condensed; striking at LARGE sizes only; fashion/beauty/luxury/editorial.
- **Newsreader** — screen-tuned serif, comfortable+professional; corporate/academic.
- **Montagu Slab** — bold slab, grounded/historical; books/history/craft.
- **Stint Ultra Expanded** — wide retro serif, statement headlines; sports/automotive/fitness.

### Mono / technical
- **IBM Plex Mono** — clean monospaced, tech-forward, 14 styles, reads at small sizes; coding/finance/productivity.

## Build decisions (not font names)
1. Default to a **sans serif** for overlay/headline text on screens — it survives small sizes best. Reach for serif/slab only for an intentional editorial or luxury tone, and only at large sizes.
2. **Go heavy.** Pick the weight for the smallest size it'll be seen at (mobile thumbnail strip), not the size in your editor. If a face only works thin, it's the wrong face here.
3. **Prefer variable / optical-sizing faces** (Jaro, Geist, Inter) when the same text reorganizes across layouts/breakpoints — adjust weight and width instead of swapping fonts.
4. **Keep the copy to 3-5 words.** "10 productivity hacks" beats "10 hacks to make you more productive." Fewer words = bigger type = legible.
5. **Match family to content tone:** blocky/condensed = energy & action; geometric sans = clean/modern; rounded = friendly; serif/slab = editorial/heritage/luxury; mono = technical.
6. **Stay consistent for brand recall** — lock one display family (+ one weight pairing) per channel/section so it's recognizable at a glance.
7. **Compose with the rule of thirds** and design/verify at the real target resolution (1280x720 for thumbnails), then check the *mobile* render — text legibility is decided on the small screen, not the desktop mock.

## Build checklist
- [ ] Headline is a heavy/bold weight, legible at the SMALLEST size it'll appear (checked on mobile, not just desktop)?
- [ ] Sans serif by default; serif/slab only chosen deliberately and only at large size?
- [ ] Overlay copy trimmed to ~3-5 high-impact words?
- [ ] Font family's personality matches the content tone (impact / clean / friendly / editorial / technical)?
- [ ] One locked display family + weight reused for brand recall across the set?
- [ ] Layout uses thirds/grid alignment, and text contrasts enough to survive a busy background?
