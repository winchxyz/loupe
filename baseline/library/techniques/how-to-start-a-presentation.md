# Opening hooks — applied patterns

Core idea: the first screen does the same job as a presentation's first 60 seconds — it sets tone, earns attention, and decides whether the rest gets read. Pick ONE hook and commit the whole opening to it; don't stack three. Below are the reusable patterns (the article's 20 ideas collapse into these), each with a "when to use."

## The hook patterns (each = a build decision for the first screen)

1. **Tension / myth-break** — open by contradicting what the visitor assumes ("Everything you know about X is wrong"). Use when your value prop reframes an old way of working. Build it as a single bold contrarian line, no sub-copy competing with it.
2. **Provoking stat / data point** — lead with one hard number that surfaces the problem, then resolve it below the fold. Use for credibility-driven or B2B sites. Visualize the number (chart/large numeral), never bury it in a paragraph; one number, not five.
3. **Engaging question** — pose a question the visitor silently answers, pulling them in. Use when the audience self-segments ("Ever shipped a deadline you regret?"). Keep it to one line; a polarizing/personal question beats a generic one.
4. **Clear unsolved problem first** — name the shared pain BEFORE the solution; validate frustration to earn trust, then promise the fix. Use for any product page — resist opening on the solution.
5. **Show the work / proof** — lead with the actual output (a real screen, demo, prototype) instead of a bio or feature list. Use for portfolios and product launches; craft speaks faster than claims.
6. **Personal "why" / anecdote** — open on the motivation or a moment of vulnerability so the visitor cares before they see how it works. Use for founder stories, mission-led brands.
7. **Famous quote as anchor** — a single well-chosen quote frames the whole thesis (e.g. Ford's "faster horses"). Use when you need to borrow an established principle as setup. One quote, attributed, large type, nothing else on screen.
8. **"Imagine…" sensory scenario** — descriptive language drops the visitor into a future state or visceral scene before the solution appears. Use when the product is abstract and needs felt stakes. Pair with imagery/motion that matches the described scene.
9. **High-impact visual / cinematic cold open** — start near-black, then reveal ONE high-quality image or short looping video that embodies the topic; the sudden contrast sets mood wordlessly. Use for hero/landing where atmosphere carries the message — this is the pattern our scroll-journey work leans on (see ties).
10. **Interactive participation** — make the first interaction the hook: a poll, a hover/scrub, a playable prototype, a "raise your hand" prompt. Use when engagement-from-frame-one matters; turns a passive viewer into an active one.
11. **Agenda / roadmap** — show a table of contents so the visitor knows the journey ahead. Use for long, dense, or multi-section experiences where orientation builds anticipation rather than spoiling it.
12. **Brand evolution / then-vs-now** — open on the journey from where the brand started to where it is now, as the transition into what's new. Use for redesign reveals, milestone launches, anniversary pages.

## Delivery rules (carry over to motion/layout, not just copy)

- **Nail the opening frame.** It's the highest-risk moment — make the first screen the most rehearsed, most polished, zero-jank part of the build.
- **Pause for effect.** Let the hook land before the next beat: a deliberate beat of stillness / whitespace / a hold in the scroll before the second element animates in. Don't fire everything at once.
- **Let visuals do the talking.** Replace dense text with one large impactful visual; the "7x7 rule" (≤7 lines, ≤7 words/line) is the ceiling for any text-heavy slide or hero.
- **One focal point.** After the hook, a single strategic pause/anchor — not competing CTAs — projects control.

## Build checklist (use when designing any opening screen)

- [ ] Exactly ONE hook pattern chosen and committed (not three stacked)?
- [ ] Does the opening name a tension/problem/number/scene BEFORE showing the solution or features?
- [ ] First screen reduced toward one focal point + minimal text (7x7 ceiling honored)?
- [ ] A deliberate beat (whitespace / scroll-hold / animation delay) lets the hook land before the next element?
- [ ] The opening frame is the most polished, jank-free moment in the build?
- [ ] If using the cinematic cold-open: starts dark/quiet, reveals one high-quality image or loop, no copy fighting it?

> Ties to our experience: 02b/TESQ is exactly pattern 9 — a cinematic cold open (water → space → sun) that earns attention before any UI appears, and pattern 11's roadmap logic maps to its sticky-pinned section sequence. The delivery rule "the opening frame must be the most polished, jank-free moment" is precisely where our PAIN.md verification-coverage gaps bite (trusting a settled frame / clipped screenshot) — verify the hero frame on every perf tier, since a janky first second wastes the hook.
