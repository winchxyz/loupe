# Thank-you card design — applied rules

Core idea: a thank-you is a TONE problem before it is a layout problem. The article sorts 49 messages into seven contexts (professional, sincere, funny, wedding, holiday, gift, teacher). Each context = a distinct register that should drive type, color, and density choices. Build the register first, then the page.

## Tone registers (pick one, then commit type + color to it)
1. **Professional** — concise, warm-but-restrained. Choose a clean grotesque or humanist sans, generous whitespace, one muted accent. Avoid slang, avoid over-explaining; one clear line of gratitude beats a paragraph.
2. **Sincere / heartfelt** — emotion-forward. Choose softer, higher-contrast type (a refined serif or a warm script for the salutation), muted/desaturated palette, room for a longer hand-written-feeling message. Be specific about the gesture; never generic, never self-centered.
3. **Funny / playful** — lighthearted. Choose a rounded or characterful display face, brighter playful color combos, a punchline as the focal line. Still land one real "thank you" — don't let the joke eat the message.
4. **Wedding** — celebratory + personal. Pair an elegant display/script headline with a readable body face; leave a photo slot (the article suggests a candid picture). Personalize per-recipient; never the same pasted note to everyone.
5. **Holiday / seasonal** — festive but legible. A seasonal display accent is fine; keep body type plain. Make it print-ready (this is the one register the article explicitly ties to printable/mailed cards).
6. **Gift acknowledgment** — short, specific, named. Layout should foreground the gift reference and leave room to sign. Always name the gift and sign off.
7. **Teacher / mentor appreciation** — respectful + specific. Clean type, room for one concrete memory; avoid an overly casual tone.

## Reusable layout patterns
- **One focal line.** Every register has a single hero sentence. Set ONE line at display scale; demote the rest. Don't give two lines equal weight.
- **Always leave a sign-off slot.** "Forget to sign your name" is called out as a failure across registers — reserve explicit space for greeting + sign-off, top and bottom.
- **Personalization is structural, not decorative.** The recurring "don't copy-paste a generic note" warning means the template must have an obvious editable region (name, the specific gift, a shared memory) — design the blank in, don't bury it.
- **Match color + font to register, not to taste.** Playful → bright combos + characterful face; sincere → muted + serif/script; professional → one muted accent + clean sans. The register dictates the palette.
- **Print path is a real mode.** When the artifact may be mailed/printed (holiday especially): design at print dimensions, CMYK-safe colors, embed/outline fonts, and verify the printable export separately from the on-screen version.

## Four-step authoring flow (reuse for any confirmation/appreciation UI)
1. Pick the template that matches the moment (occasion + recipient set the register).
2. Write the message: greeting → one specific thank-you → warm sign-off.
3. Customize: colors, font pairing, image — small changes, big impact.
4. Ship to the right channel: print/download OR digital — and verify that exact channel.

## Build checklist (tick while applying)
- [ ] One tone register chosen, and type + color committed to it (not mixed registers)?
- [ ] Exactly one focal/hero line at display scale; everything else demoted?
- [ ] Explicit editable slots for the specific detail (name / gift / memory) so it can't read as a generic blast?
- [ ] Greeting AND sign-off space reserved — name can actually be signed?
- [ ] Body copy legible: the decorative/script face is used for the headline only, never the paragraph?
- [ ] If it may be printed: print dimensions, bleed/CMYK, and the printable export verified separately from the screen view?
