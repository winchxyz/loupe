# Brand style guide — applied rules

Core idea: a style guide is the single source of truth that keeps every channel (site, app, email, social, video) on-brand. For our builds it is the upstream doc that feeds DESIGN.md — decide the brand once, then every section inherits it instead of re-improvising. Spin up v1 the moment you have a handful of assets; treat it as living, not a one-time artifact.

## The 5 elements to lock (each = a build decision)
1. Brand story — write down WHY/HOW the product addresses the user's problem in plain, authentic language. Choose this before copy; it sets the emotional register the hero and section narrative must hit. Avoid jargon and slogans that don't trace back to mission/values.
2. Target audience — define the persona before picking type, color, or tone. Let their needs drive identity choices; do NOT design to your own taste in a vacuum. If unsure, do quick user research rather than guess.
3. Visual identity — specify the full token set up front: typography + font sizes, logo usage rules, imagery direction, iconography, and color in RGB + HEX + CMYK. This IS the design-system seed; codify it so cohesive assets fall out automatically rather than being eyeballed per screen.
4. Brand voice — pin the personality to a tight adjective set (e.g. "modern, clever, and brave, yet caring"). Choose tone-per-channel up front (casual for social, professional for formal contexts) so microcopy doesn't drift screen to screen.
5. Writing guidelines — set readability level, grammar rules, and an approved/banned word list. Apply these to every label, button, and body block, not just marketing pages.

## Process rules (do this, in order)
6. Outline first — draft the chapters (story, audience, visual identity, voice, writing) and build a real table of contents so any token or rule is findable in seconds. Findability is a feature: a guide nobody can search gets ignored.
7. Capture, don't pad — keep the guide tight (target ~25–35 pages); link out to deeper resources instead of inlining everything. Bake accessibility specs into the visual + copy components here, not as an afterthought.
8. Revise for clarity — run it past stakeholders inside and outside the brand team; cut for simplicity before "shipping" it.
9. Ship + socialize — put it where everyone (staff and freelancers) can actually reach it; an unread guide buys nothing.

## Pro rules that change output
10. Make it interactive — let people submit questions/requests against the guide so it stays a tool, not a PDF tombstone.
11. Update on a cadence — revisit every ~6–12 months (and seasonally as the brand evolves); a stale guide silently licenses off-brand drift.
12. Show dos AND don'ts — pair every rule with a concrete in-context example of right vs. wrong. Examples implement faster than prose and pre-empt the predictable mistake.

## Build checklist (use when starting or reviewing a brand build)
- [ ] Brand story + persona written down BEFORE any type/color/tone choice?
- [ ] Visual identity captured as real tokens (type scale, logo rules, color in HEX/RGB/CMYK) feeding DESIGN.md?
- [ ] Voice fixed to an adjective set, with per-channel tone noted?
- [ ] Writing rules (readability, grammar, approved/banned words) applied to UI labels, not just marketing copy?
- [ ] Accessibility specs baked into the components, not bolted on later?
- [ ] Each non-obvious rule paired with a concrete do/don't example?
- [ ] Guide findable (table of contents) and dated for its next review?
