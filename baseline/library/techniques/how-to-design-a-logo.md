# How to design a logo — applied build decisions

Core idea: a logo is a brand's identity distilled to one scalable mark ("silent salesman"). Pick a logo TYPE, then drive color, type, and shape from brand values — not aesthetics in a vacuum. Build it to stay legible from favicon to hero.

## Pick the logo TYPE first (it constrains everything after)
1. **Wordmark** — name-only, typography-led. Choose when the name is short and distinctive and you want it to BE the mark (e.g. Coca-Cola). Type carries 100% of the brand here.
2. **Letterform** — single glyph. Choose only for already-recognized brands that can survive on one letter (Facebook "f", Pinterest "P"). Risky for unknowns.
3. **Monogram** — initials, simple and compact. Choose when the full name is long but the initials are ownable (CNN, H&M, IBM).
4. **Abstract** — invented shape/symbol, no literal depiction. Choose for a modern, differentiated feel where no existing object fits (Pepsi, Audi).
5. **Combination** — text + symbol locked together. Default safe choice: the symbol works small, the wordmark spells out the name (Airbnb, Dropbox, Doritos).
6. **Emblem** — text contained inside a shape/badge. Choose for traditional, heritage, or institutional tone (Starbucks, BMW). Avoid when you need it tiny — emblems lose detail at small sizes.

## Color: theory + psychology, not taste
7. Use color THEORY for the mechanics — contrast for legibility, and a defined scheme (complementary or analogous) for harmony. Never let the mark fail contrast against its background.
8. Use color PSYCHOLOGY for the meaning — pick the hue for the feeling it triggers (UPS brown = reliability/security). Decide the emotion first, then the hex.

## Typography (matters most for wordmark / letterform / monogram)
9. **Display** — decorative, distinctive; the common logo workhorse when you want character.
10. **Serif** — timeless, professional, authoritative-classic.
11. **Sans serif** — modern, minimal, maximally readable; safe default for tech/clean brands.
12. **Monospace** — blocky, even spacing; retro/vintage/technical vibe.
13. **Slab serif** — thick strokes, powerful and authoritative; use to grab attention.
14. **Script** — handwriting feel; elegant/creative, but stress-test readability at small sizes.
    Rule: the logo font must be readable AND express the personality — never pick one without the other.

## Shapes carry meaning — choose to match values
15. **Basic geometric** — circle = unity/community; square/rectangle = reliability/security; use for trust-forward brands (Dropbox's geometry = professionalism).
16. **Organic** — flowing lines/curves = movement, direction, fluidity (Nike swoosh = motion/energy).
17. **Abstract** — combined geo shapes/lines for a unique, ownable form when basic shapes feel generic.

## Mockup composition rules
18. **White space** — keep generous empty space between elements; clutter kills balance.
19. **Visual hierarchy** — use placement + color contrast so the most important element wins the eye.
20. **Alignment** — fix all elements to a common baseline/grid; misalignment reads as broken even when nothing else is.

## Finalize for real use
21. **Ship a variation family** — full lockup (name + mark) for wide contexts, mark-only for tiny ones (favicon, avatar, social). Don't force one size to do every job.
22. **Right file format per use** — SVG for web/screen (scales, stays crisp); PNG/JPG for raster/print contexts.
23. Validate against the good-logo test: simple, timeless, unique, relevant to audience/industry, readable at any size.

## Build checklist
- [ ] Logo TYPE chosen on purpose and justified by brand values (not picked by look)?
- [ ] Color decided from BOTH theory (contrast/scheme) and psychology (emotion), and passes contrast on its background?
- [ ] Font is both readable AND on-personality — verified at small size, not just at hero size?
- [ ] Elements aligned to one baseline/grid, with deliberate white space and a clear focal element?
- [ ] Variation family exists: full lockup + mark-only for favicon/avatar?
- [ ] SVG used for the on-site mark (crisp at every scale)?
- [ ] Passes the 5 tests: simple, timeless, unique, relevant, readable?

Rule #21 (mark-only variant for favicon/social) is the scalability case our clipped-screenshot verification gap would otherwise miss.
