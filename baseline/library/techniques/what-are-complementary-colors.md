# Complementary colors — applied rules

Core idea: complementary = hues opposite on the wheel. Side by side they trigger *simultaneous contrast* (each looks brighter/more saturated), so they're the strongest tool for pulling the eye to ONE thing. Power = restraint: a full-bleed complementary pair vibrates and tires the eye; use it for the accent, not the field.

## The rules (each = a build decision)

1. Pick the scheme by JOB. Need a focal point / CTA / state signal? Reach for a complementary (opposite) accent against your base. Need calm/cohesion? Don't — use analogous instead. Complementary is a spotlight, not a wallpaper.

2. Use the right wheel for the medium. Screens are RGB (additive): the digital complement pairs are red↔cyan, green↔magenta, blue↔yellow. Don't pick complements off the painter's RYB wheel (red↔green, blue↔orange, yellow↔purple) when you're building for screen — the opposite hue is literally different. CMYK (print) gives magenta↔green, yellow↔blue, cyan↔red.

3. Make the accent the minority. Apply the bold complementary hue only to the most important elements — primary button, active state, key number, error. Let the base hue (or neutral) own the field. Think Google's logo: a 4-color tetrad used sparingly on stark white, not edge-to-edge.

4. Soften with split-complementary when the direct pair is too loud. Keep your base, but instead of its exact opposite use the two hues either side of that opposite. Example: base = blue → use red-orange + yellow-orange instead of pure orange. You keep the lively interplay, lose the eye-strain tension.

5. Tune tints and shades, don't fight raw hues. If a pair clashes or buzzes, push one toward a lighter tint (+white) or darker shade (+black) and drop saturation. Two fully-saturated complements at equal value = the worst-case vibration; offsetting value/saturation fixes it while keeping the contrast.

6. Pair warm against cool for max pull. The most energetic complement is a warm vs cool split — e.g. warm red-orange against cool blue-green. Great for injecting life into a type-heavy page or an otherwise monochromatic layout.

7. Lean on the contrast for hierarchy AND legibility. The same opposite-hue contrast that grabs attention also lifts text off its background — use it to push key text/controls toward accessible contrast, not just decoration. Still verify the actual ratio (don't assume hue contrast = passing contrast).

8. Let the pair carry meaning. Complementary accents read as semantic opposites for free: correct/incorrect, on/off, primary/secondary (Duolingo green + red accents; Lakers purple + yellow; Firefox orange + blue). Map the contrast to a real state, don't just decorate.

## Build checklist (use when picking an accent/palette)

- [ ] Scheme chosen for a reason — complementary because something must POP, not by default?
- [ ] Complement taken from the RGB wheel (screen), not RYB? (blue's screen-complement is yellow, not orange)
- [ ] Accent hue confined to the minority of pixels; base/neutral owns the field?
- [ ] If the direct pair buzzes: switched to split-complementary OR offset value/saturation via tints/shades?
- [ ] Warm-vs-cool used where the layout needs energy (type-heavy / monochrome section)?
- [ ] Actual text/control contrast ratio verified, not assumed from "they're opposites"?
- [ ] Each complementary use maps to a real focal point or state, not blanket decoration?
