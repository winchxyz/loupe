# Split sections without the template disease

tags: split, two-column, text-image, alternating, saas, landing, rhythm

**The disease (audited 2026-07-03):** generated sites default to a spine of 4-6 identical
alternating 50/50 text+visual splits at one constant rhythm — the single strongest
"AI template" tell. A split is a fine tool; a CHAIN of them is the failure.

## Rules

1. **Max TWO true splits per page.** Everything else must be another shape (bento,
   diagram, poster, rail, table, full-bleed band).
2. **Never 50/50 twice.** Vary the ratio: 1.2fr/0.8fr, then 0.9fr/1.1fr. The gutter and
   vertical offsets vary too (one media block pulled up -40px across the section rule).
3. **Break the wrap once.** At least one section escapes the container: a full-bleed image
   band, an inverted slab, or a marquee — the page must not live entirely inside one max-width.
4. **Modulate rhythm.** Section paddings are not equal: a dense zone (bento/table) follows
   an airy one (poster/statement). One shared spacing token, multiplied ×0.6 / ×1 / ×1.4.
5. **The visual side must be EARNED:** a drawn micro-mock, a real photo with a caption bar,
   a data readout — never an abstract placeholder shape.

## A healthy page spine (example melody)

hero (asymmetric split + vignette) → marquee band → diagram section → bento →
ONE split (1.2/0.8) → poster/inverted CTA → footer.
Seven shapes, one of them a split. Compare: hero → split → split → split → split → cards → CTA.

## Checklist before shipping a page

- [ ] Count true splits: ≤2?
- [ ] Does anything bleed past the container?
- [ ] Are any two adjacent sections the same shape? (If yes — merge or reshape one.)
- [ ] Cover the copy: can you still tell sections apart by silhouette alone?
